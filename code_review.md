# code_review.md

When reviewing code for this project, check:

1. Does this story satisfy its acceptance criteria?
2. Is the code simple and readable?
3. Does it avoid hindsight leakage?
4. Does it use cached API data where appropriate?
5. Are output files saved in the right place?
6. Are there validation checks?
7. Is the notebook still readable?
8. Are helper functions moved to `src/fpl_retro/` when they become reusable?
9. Did the agent avoid unrelated refactoring?
10. Is `STATUS.md` updated?
11. Is `KANBAN.md` updated?
12. Are task checkboxes in `BACKLOG.md` updated?
13. Are important design decisions recorded in `DECISION_LOG.md`?
